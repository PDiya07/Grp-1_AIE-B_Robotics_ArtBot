ArtBot: Autonomous AI-Driven Robotic Drawing System

4-DOF Serial Manipulator Google Gemini + Python Arduino UNO + PCA9685 Drawing Accuracy ≈ ±2 mm
01 Introduction

ArtBot is an intelligent robotic drawing system that integrates artificial intelligence, mathematical modeling, and embedded control systems to translate natural language prompts into physical artwork.

What makes ArtBot different Unlike conventional robotic systems that require predefined motion commands, ArtBot enables natural language interaction, automated coordinate generation, real-time motion computation, and physical reproduction of AI-generated shapes.

The system demonstrates how computational intelligence can be embodied into mechanical motion using kinematic transformations and real-time control loops.

Key Capabilities
  • Natural Language Interaction — user simply describes the shape in plain English (e.g. "Draw a square")
  • Automated Coordinate Generation — AI converts the description into structured 2D Cartesian (x, y) vertex data
  • Real-Time Motion Computation — inverse kinematics is solved on the fly for every interpolated path point
  • Physical Reproduction of AI-Generated Shapes — the robotic arm physically draws the result on paper with a pen
02 Motivation

The evolution of robotics has primarily focused on industrial automation, repetitive task execution, and precision manufacturing. However, creative robotics remains a largely underexplored domain.

Core Concept ArtBot explores the concept of embodied AI — where digital intelligence produces tangible, physical outputs that exist in the real world.
Three Key Research Directions
  • Bridging generative AI with physical robotic systems — closing the gap between language models and mechanical actuators.
  • Applying inverse kinematics in real-world hardware — validating mathematical IK models on a servo-based 4-DOF arm.
  • Creating a cost-effective educational robotics platform — demonstrating AI-robotics integration with accessible, low-cost components.

06 Path Planning & Trajectory Generation

The AI returns only the key vertices of a shape. Direct joint movement between vertices — without intermediate waypoints — causes curved trajectories because rotational joints move in arcs, not straight lines.

The Core Problem When a robotic joint moves from angle A to angle B, the end-effector traces a circular arc in Cartesian space. For straight-line drawing, this must be corrected.
6.1 Solution: Linear Interpolation

Dense intermediate points are generated between every pair of vertices using linear interpolation. This converts each straight line segment into N micro-steps, each small enough that the arc deviation is negligible.

// Linear interpolation between two shape vertices // P₁(x₁, y₁) → P₂(x₂, y₂) For i = 0 to N: xᵢ = x₁ + i × ((x₂ − x₁) / N) yᵢ = y₁ + i × ((y₂ − y₁) / N) Where: N = number of intermediate steps (controls smoothness) xᵢ = interpolated x coordinate at step i yᵢ = interpolated y coordinate at step i
✗ Without Interpolation
  • Only vertex coordinates are commanded
  • Joint space motion produces curved arcs
  • Straight lines appear bent or curved on paper
  • Geometric accuracy is compromised
✓ With Linear Interpolation
  • Dense micro-points generated per segment
  • Each step is a tiny, nearly-straight movement
  • Smooth straight-line drawing output
  • Improved geometric accuracy
07 Mathematical Foundation of Inverse Kinematics

Inverse Kinematics (IK) is the mathematical process that determines the joint angles required to position the end-effector (pen tip) at a desired target position in 3D space. Rather than manually controlling each motor, the target position is specified and the angles are computed automatically.

The IK Problem Statement Given a target 3D position P(x, y, z), compute joint angles θ₁ (Base), θ₂ (Shoulder), θ₃ (Elbow), and θ₄ (Wrist) such that the pen tip reaches P exactly.
7.1 Base Rotation — θ₁

The base joint rotates the entire arm horizontally to point toward the target. The atan2 function is used to ensure correct quadrant handling. An offset of +90° aligns the coordinate frame with the robot's physical orientation.

θ₁ = atan2(y, x) + 90° Where: atan2(y, x) → computes angle from origin to target in XY plane +90° → coordinate frame alignment offset
7.2 Horizontal & 3D Distance

Before computing shoulder and elbow angles, two intermediate distances are calculated. These form the sides of the geometric triangle used in the law-of-cosines solution.

// Horizontal reach of the arm in XY plane: r = √(x² + y²) // 3D distance from shoulder joint to target point: d = √(r² + z²) Note: d is the hypotenuse of the triangle formed by link L₁, link L₂, and the shoulder-to-target line.
7.3 Elbow Angle — θ₃

The elbow angle is solved directly using the Law of Cosines applied to the triangle formed by link L₁, link L₂, and the shoulder-to-target distance d.

cos(θ₃) = (L₁² + L₂² − d²) / (2 · L₁ · L₂) θ₃ = arccos( (L₁² + L₂² − d²) / (2 · L₁ · L₂) ) Where: L₁ = upper arm link length (shoulder to elbow) L₂ = forearm link length (elbow to wrist) d = shoulder-to-target distance (computed above)
7.4 Shoulder Angle — θ₂

The shoulder angle combines two components: α, the elevation angle to the target, and β, the internal angle at the shoulder vertex of the link triangle.

α = atan2(z, r) // elevation to target β = arccos( (L₁² + d² − L₂²) / (2 · L₁ · d) ) // internal angle θ₂ = α + β
7.5 Wrist Angle — θ₄

The wrist angle is derived from a constraint: the pen must remain perpendicular to the drawing surface at all times. This is enforced by computing the remaining angle needed to keep the tool orientation fixed.

θ₄ = 180° − (θ₂ + θ₃) This ensures the pen tip is always perpendicular to the paper, regardless of the shoulder and elbow configuration. (Total angle constraint: θ₂ + θ₃ + θ₄ = 180°)

08 Control Loop & Command Execution

The Python program acts as the primary controller, orchestrating the entire drawing sequence. It bridges the AI-generated coordinate data with the physical hardware in real time.

8.1 Control Sequence
1

Generate interpolated waypoints

Software

Apply linear interpolation to the AI-provided vertex list, creating N dense (x, y) coordinate points for each line segment of the shape.

2

Compute IK for each waypoint

Software

For each interpolated point (x, y, z), apply the inverse kinematics equations to compute the four joint angles θ₁, θ₂, θ₃, θ₄.

3

Convert angles to servo commands

Software

Format the four computed angles into a comma-separated command string matching the Arduino communication protocol.

// Command format: Base,Shoulder,Elbow,Wrist // Example output: cmd = f"{int(t1)},{int(t2)},{int(t3)},{int(t4)}\n" # → "90,110,45,25\n"
4

Transmit via serial communication

Hardware

Each command string is transmitted over USB-UART serial at 115200 baud from Python to the Arduino. The Arduino parses the packet and forwards PWM instructions to the PCA9685 servo driver.

import serial, time ser = serial.Serial('COM3', 115200, timeout=1) for point in interpolated_path: t1, t2, t3, t4 = inverse_kinematics(point) cmd = f"{t1},{t2},{t3},{t4}\n" ser.write(cmd.encode()) time.sleep(0.05) # allow servo settling time

11 Challenges & Solutions

Three key engineering challenges were encountered and resolved during development and testing of the ArtBot system.

Challenge Root Cause Engineering Solution
⚠ Servo Instability Voltage drop on shared Arduino 5V rail during high-torque servo movement caused erratic behavior and random microcontroller resets. ✓ Implemented a dedicated 5V 5A switched-mode power supply (SMPS) with an isolated servo power rail. PCA9685 offloads all PWM generation from Arduino.
⚠ Curved Drawing Lines Rotational joint motion between only the vertex endpoints produces circular arcs in Cartesian space instead of straight lines on paper. ✓ Applied linear interpolation between every pair of vertices to generate N dense micro-waypoints. Straight-line motion approximated at each small step.
⚠ Pen Height Calibration Mechanical assembly tolerances caused inconsistent pen-to-paper contact — too high produced no ink, too low caused dragging and smearing. ✓ Introduced an adjustable Z-offset parameter in the Python software. Allows fine-tuning of pen contact height without any physical hardware changes.
